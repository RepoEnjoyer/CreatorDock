# Roadmap

CreatorDock 1.0 focuses on a dependable single-device workflow. Future work should preserve its local-first, no-account core.

## Near term

- Add optional project archiving and an archive view.
- Add calendar navigation by keyboard and improved small-screen day summaries.
- Add selective export for one project or one template.
- Add browser-level accessibility tests for project creation and status changes.
- Add install guidance that responds to each browser's PWA capabilities.

## Later

- Add opt-in encrypted workspace files using a user-supplied passphrase.
- Add dependency-free Markdown export for project briefs.
- Add local-only project relationships, such as a series containing several pieces.
- Add an optional local reminder format that can be imported into calendar applications.
- Evaluate a desktop wrapper only if it improves file ownership without compromising the web app.

## Explicitly out of scope

- Mandatory accounts or hosted synchronization
- Advertising, tracking, or engagement scoring
- Paid APIs required for core features
- Automated publishing that requires users to hand credentials to CreatorDock
- Social feeds, follower counts, or vanity metrics

Roadmap entries are proposals, not promises. Open a feature request with a concrete creator workflow and privacy impact to help shape priorities.
